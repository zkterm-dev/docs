## zkID

<Card title="zkID Documentation" icon="fingerprint" href="/zk-ecosystem/zkid">
  Privacy-preserving universal identity
</Card>

**Key Features:**
- `did:zk:[12 alphanumeric chars]` universal DID format
- Client-side Ed25519 proof generation
- Dual-chain on-chain verification (Solana Mainnet + Starknet Sepolia)
- Cairo smart contract for zkID proof storage

```bash
zkid create           # Create new zkID
zkid verify           # Verify your identity
zkid status           # Check on-chain status
```

## zkCompression

<Card title="zkCompression Documentation" icon="compress" href="/zk-ecosystem/zkcompression">
  Light Protocol integration for Solana
</Card>

**Key Features:**
- Compress SPL tokens to reduce storage costs by 1000x
- Decompress tokens back to regular SPL tokens
- Full terminal and API integration
- Automatic retry polling for RPC indexer sync

```bash
compress <mint> <amount>     # Compress tokens
decompress <mint> <amount>   # Decompress tokens
zk tokens                    # View all tokens (regular + compressed)
```

## zkStorage

<Card title="zkStorage Documentation" icon="hard-drive" href="/zk-ecosystem/zkstorage">
  Encrypted decentralized file storage
</Card>

**Key Features:**
- Client-side AES-256-GCM encryption
- IPFS storage via Pinata
- Groth16 zkSNARK proofs for password verification
- SHA-256 checksum validation

```bash
zk storage upload <file>     # Upload encrypted file
zk storage download <cid>    # Download and decrypt
zk storage list              # List your files
zk storage verify <cid>      # Verify with ZK proof
```

## zkAuth

<Card title="zkAuth Documentation" icon="key" href="/zk-ecosystem/zkauth">
  Passwordless authentication with STARK proofs
</Card>

**Key Features:**
- Web3Auth social login (Google, GitHub, X)
- No passwords, no seed phrases
- STARK proofs on Starknet for verification
- Quantum-resistant cryptography

```bash
zkauth login google          # Login with Google
zkauth login github          # Login with GitHub
zkauth status                # Check auth status
```
